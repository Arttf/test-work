# AI Strategy

## Component Structure

- Не размещать много шаблонов или крупных компонентов в одном файле.
- Файл `index.tsx` или `index.ts` должен быть точкой входа или реэкспорта, а не контейнером для нескольких больших шаблонов.
- Если в компоненте появляется вторая крупная визуальная секция или отдельный шаблон, его нужно выносить в отдельный файл.
- Один файл = одна основная ответственность.
- Вложенные части, которые можно переиспользовать или читать отдельно, должны быть вынесены в самостоятельные компоненты.

## Rules For UI Files

- `Base`-компоненты хранить отдельно от их вариаций.
- Варианты кнопок, полей, карточек и других элементов строить поверх базового компонента, а не смешивать всю логику в одном месте.
- Стили и переменные держать отдельно, если это упрощает масштабирование и поддержку.

## Example

Плохо:

- `index.tsx`, внутри которого лежат `SidebarLayout`, `SidebarToggle`, `SidebarMenu`, `SidebarFooter`.

Хорошо:

- `index.ts` только экспортирует публичное API.
- `SidebarLayout.tsx` содержит только layout.
- `SidebarToggle.tsx` содержит только toggle.
- `SidebarMenu.tsx` содержит только меню.
- `SidebarFooter.tsx` содержит только footer.

## Goal

Главная цель: чтобы AI и человек могли быстро читать проект, безопасно менять код и без боли добавлять новые состояния, шаблоны и варианты компонентов.
